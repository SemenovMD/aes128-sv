# AES128 Decryptor и Encryptor

Данный модуль представляет собой высокопроизводительную реализацию `AES-128 дешифратора и шифратора` на `SystemVerilog`. Модуль поддерживает параллельную обработку данных через кластер ядер, который интегрирован в `AXI-Stream` интерфейс и оптимизирован для работы на FPGA.

## Особенности
- **Параллельная архитектура** - конвейерные ядра AES-128
- **Высокая пропускная способность** - 1 операция дешифрования за такт (после начальной задержки)
- **AXI-Stream интерфейс** - стандартизированный потоковый интерфейс
- **Статические ключи** - предварительно вычисленные раундовые ключи
- **Полный конвейер** - минимальная задержка между операциями
- **Оптимизирован для FPGA** - эффективное использование ресурсов

## Архихитектура
AXI-Stream Input → Round-Robin Dispatcher → N×AES128 Cores → Round-Robin Arbiter → AXI-Stream Output


## Параметры
- `CORE` - количество параллельных ядер AES-128 (по умолчанию: 44)
- `KEY` - массив раундовых ключей для дешифрования (11×128 бит)

## Производительность
- **Латентность**: 42 такта на операцию
- **Пропускная способность**: 1 блок/такт (после заполнения конвейера)
- **Тактовая частота**: до 333 МГц (при CLK_PERIOD=3ns)
- **Производительность**: ~116 миллионов операций/сек

## Симуляция
Для симуляции используется `QuestaSim`. Убедитесь, что путь к исполняемому файлу `vsim` добавлен в файл `bashrc`.

Для запуска симуляции `AES128 Decryptor CORE` выполните команду:
```sh
make sim_dec_42_cycle
```
Для запуска симуляции `AES128 Decryptor CLUSTER CORE` выполните команду:
```sh
make sim_dec_1_cycle
```
Для запуска симуляции `AES128 Encryptor CORE` выполните команду:
```sh
make sim_enc_42_cycle
```
Для запуска симуляции `AES128 Encryptor CLUSTER CORE` выполните команду:
```sh
make sim_enc_1_cycle
```
Для очистки файлов, полученных при симуляции выполните команду:
```sh
make clean
```
