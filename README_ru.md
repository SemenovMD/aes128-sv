# AES-128 Core и Pipeline

Модуль представляет собой одноядерную или конвейерную архитектуру `AES-128` на `SystemVerilog` с интегрированным `AXI-Stream` интерфейсом, оптимизированная для развертывания на FPGA платформах `Xilinx`.

## Особенности

- Одноядерная архитектура.
- Конвейерная архитектура.
- Блок расширения ключей.
- Обмен данными через `AXI-Stream` интерфейс.
- Поддержка шифрования и дешифрования.
- Задержка - 43 такта

## Параметры IP

- `START_KEY` - начальный ключ AES-128. По умолчанию: `128'h2B7E151628AED2A6ABF7158809CF4F3C`.
- Раундовые ключи генерируются автоматически в модуле `key_expansion`.

## Производительность и утилизация

Метрики имплементации для `Xilinx Kintex-7 xc7k325tfbg676-2`.

### AES128 Decryptor

```sh
| Mode     | LUT    | FF     | BRAM | Frequency (MHz) |
|----------|--------|--------|------|-----------------|
| Core     | 1178   | 270    | 0    | 300+            |
| Pipeline | 8447   | 5300   | 0    | 300+            |
```

### AES128 Encryptor

```sh
| Mode     | LUT    | FF     | BRAM | Frequency (MHz) |
|----------|--------|--------|------|-----------------|
| Core     | 1066   | 271    | 0    | 300+            |
| Pipeline | 6722   | 4980   | 0    | 300+            |
```

### AES128 Key Expansion
```sh
| LUT    | FF     | BRAM | Frequency (MHz) |
|--------|--------|------|-----------------|
| 2392   | 1709   | 0    | 300+            |
```

Для предотвращения использования BRAM и реализации памяти на основе распределённых регистров (distributed ROM), добавьте следующее ограничение в ваш XDC файл:

```tcl
set_property rom_style distributed [get_cells -hier -filter {NAME =~ **}]
```

Это ограничение гарантирует, что все таблицы подстановки (S-box) будут реализованы на LUT, а не на BRAM блоках.

## Симуляция

Для симуляции используется `QuestaSim`. Убедитесь, что путь к исполняемому файлу `vsim` добавлен в файл `bashrc`.

Для запуска симуляции `AES128 Decryptor Core` выполните команду:
```sh
make sim_dec_core
```
Для запуска симуляции `AES128 Decryptor Pipeline` выполните команду:
```sh
make sim_dec_pipeline
```
Для запуска симуляции `AES128 Encryptor Core` выполните команду:
```sh
make sim_enc_core
```
Для запуска симуляции `AES128 Encryptor Pipeline` выполните команду:
```sh
make sim_enc_pipeline
```
Для очистки файлов, полученных при симуляции выполните команду:
```sh
make clean
```

## Примеры python скриптов

### `calc_dec`
Расчет операции `дешифрования AES128` по раундам.

### `calc_enc`
Расчет операции `шифрования AES128` по раундам.

### `gen_tables`
Генерация тестовых данных `ciphertext.txt` и `plaintext.txt` для верификации в testbench.

## Документация

### `aes128_core_top`
Верхнеуровневый модуль, объединяющий одноядерные блоки шифрования и дешифрования с блоком расширения ключей.

### `aes128_enc_core`
Базовое ядро шифрования AES-128.

### `aes128_dec_core`
Базовое ядро дешифрования AES-128.

### `aes128_pp_top`
Верхнеуровневый модуль, объединяющий конвейерные блоки шифрования и дешифрования с блоком расширения ключей.

### `aes128_enc_pp`
Конвейерное ядро шифрования AES-128.

### `aes128_dec_pp`
Конвейерное ядро дешифрования AES-128.

### `key_expansion`
Модуль расширения ключа. Генерирует все 11 раундовых ключей из начального 128-битного ключа.

### `tb_aes128_enc`
Тестбенч для проверки работы модулей шифрования. Генерирует тестовые векторы из файлов `plaintext.txt` и `ciphertext.txt`, выполняет шифрование и сравнивает результаты с ожидаемыми значениями.

### `tb_aes128_dec`
Тестбенч для проверки работы модулей дешифрования. Генерирует тестовые векторы из файлов `ciphertext.txt` и `plaintext.txt`, выполняет дешифрование и сравнивает результаты с ожидаемыми значениями.
